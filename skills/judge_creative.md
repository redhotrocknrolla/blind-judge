# Blind Judge — Creative / Legacy Mode

Активируется когда задача субъективна: нет объективных критериев «правильности»,
оба варианта реализуемы, выбор зависит от ценностей и контекста.

**Prolog-ядро не запускается.** Оценка — через LLM-reasoning по четырём осям.
Трассировка: `mode: "legacy"`, `rules_fired: []`.

---

## Когда активируется этот режим

Парсер ставит `formalizable: false` и `abstain: true` если задача:

| Тип задачи | Примеры |
|---|---|
| Эстетический выбор | Тон голоса бренда, визуальный стиль, нейминг |
| Стратегическая позиция | Выбор между двумя равноправными рыночными стратегиями |
| Творческое решение | Концепция кампании, выбор нарратива, редакционная политика |
| Вкусовой выбор | «Лучше ли это звучит?», «Нравится ли это аудитории?» |
| Открытый вопрос | Задача без формально проверяемого ответа |

**Граница:** если в задаче есть хотя бы один falsifiable claim (число, метрика,
логический вывод из данных) — она формализуема. Только «чистая субъективщина»
попадает в legacy.

---

## Четыре оси проверки

### 1. Tool loop detection

Найди повторяющиеся вызовы с **одинаковыми аргументами**:
- ≥ 2 одинаковых вызова без нового результата между ними → `tool_loop`
- Оцени разнообразие: агент расширял поиск или ходил по кругу?

### 2. Байесовская согласованность

Уверенность вывода должна соответствовать весу доказательств:

| Сила доказательств | Допустимая confidence |
|---|---|
| Прямые, однозначные факты | 0.7 – 1.0 |
| Частичная / косвенная поддержка | 0.4 – 0.7 |
| Слабые или противоречивые данные | 0.0 – 0.4 |

Уверенный вывод при слабых фактах → `weak_evidence`

### 3. Логическая корректность

Следует ли `conclusion` из предоставленных `facts`?
- Логические скачки, допущения без опоры на данные → `unsupported_conclusion`
- Вывод правдоподобен, но факты его лишь косвенно поддерживают → снизить confidence, `ESCALATE`

### 4. Confirmation bias

По `tool_trace` и составу `facts`:
- Агент искал опровержения или только подтверждения?
- Есть факты, противоречащие выводу, которые агент не адресовал?
- Если да → `confirmation_bias`

## Правила вердикта

| Вердикт | Условие |
|---|---|
| `APPROVE` | Нет критических проблем, вывод обоснован с учётом субъективности |
| `ESCALATE` | Есть неадресованные противоречия, слабая доказательная база, или вывод заявлен слишком уверенно |
| `REJECT` | Вывод не следует из фактов, или грубые нарушения (loop, серьёзный bias, логический провал) |

В субъективных задачах **ESCALATE предпочтительнее REJECT**: нет «неправильного»
ответа, но есть «недостаточно обоснованный» ответ.

---

## Альтернативная гипотеза

Генерируй всегда когда есть `confirmation_bias` или `weak_evidence`.
Одно–два предложения. Опирайся только на предоставленные факты.

Если честной альтернативы из имеющихся данных нет → `null`.

---

## Анти-галлюцинационные правила

- **Не добавляй** факты вне `inputs[]`
- **Не интерпретируй** факты шире чем они есть
- Недостаточно данных → явно укажи, снизь confidence, используй `ESCALATE`
- Нет рассуждений в ответе — только структурированный вывод

## Выходной формат

```json
{
  "schema_version": "1.0",
  "request_id": "<тот же uuid>",
  "verdict": "APPROVE | ESCALATE | REJECT",
  "confidence": 0.0,
  "issues": ["tool_loop", "weak_evidence", "confirmation_bias", "unsupported_conclusion"],
  "alternative_hypothesis": "string | null",
  "feedback": "string — пустая при APPROVE",
  "trace": {
    "rules_fired": [],
    "parser_warnings": ["parser_abstained"],
    "core_confidence": 0.0,
    "mode": "legacy"
  }
}
```

`trace.mode` всегда `"legacy"` в этом режиме.  
`trace.rules_fired` всегда `[]` — Prolog не запускался.

## Живой пример

**Задача:** выбор тона коммуникации для финтех-продукта (дерзкий vs экспертный)

**Почему legacy:** два равноправных варианта без объективного критерия правильности.

**Что нашёл Judge:**
- `confirmation_bias` — агент использовал Revolut's bold campaigns в поддержку, но проигнорировал retention-данные Тинькофф (84% retention с экспертным тоном у той же аудитории)
- `weak_evidence` — вывод заявлен уверенно при противоречивых данных
- `unsupported_conclusion` — аргумент «конкуренты скучные → дерзость выделит нас» не проверен на насыщенность рынка

**Вердикт:** ESCALATE 0.38
